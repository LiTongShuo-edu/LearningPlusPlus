# 调试与验证说明

## 构建方式

`＊` 复现案例与 `^` 岗位扩展示例统一按 C++17 组织，并由顶层 `CMakeLists.txt` 管理。

```powershell
cmake -S . -B build
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

无后缀的历史代码不纳入默认构建门禁，以免为了构建结果改变早期源码。其独立编译结果见 [`手写旧作编译记录.md`](./手写旧作编译记录.md)。

## 命名与 Windows 构建

`^` 是公开目录用于标明岗位定向扩展示例的字符，也是 Windows 命令解释器中的转义字符。链接器直接接收含 `^` 的中间路径时会丢失该字符，因此 CMake 配置阶段将三个 `^` 源文件复制到 `build/marked_sources/` 的构建临时路径，再进行编译。该处理不会更改公开源码或来源标识，生成目录也被忽略发布。

## 调试建议

1. 在菜单分支入口放置断点，观察选择值是否进入预期流程。
2. 在数组访问或查找循环中观察下标与容器大小，防止越界。
3. 在指针与动态内存示例中查看指针值、空指针分支和释放后的使用风险。
4. 在职工管理案例中检查文件打开是否成功，并对保存前后的记录数量做比对。

## 已知限制

- 示例为控制台教学代码，不包含输入格式的完整防御性校验。
- 职工管理使用文本文件保存基础记录，不覆盖并发写入、权限异常或数据迁移等工程场景。
- 旧作中的原始编译错误被如实保留，不计入新增代码的构建结果。
